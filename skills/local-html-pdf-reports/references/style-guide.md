# Print-Friendly Report Style

Use this baseline only when the repository has no established visual language.

## Tokens

- Paper: warm white or white.
- Ink: near-black.
- Muted text: accessible medium gray.
- Accent: one saturated color used for rules, tags, and key metrics.
- Borders: thin neutral lines; avoid heavy shadows.

## Components

- Kicker, direct headline, short deck, and verdict/status stamp.
- Metric strip for four to six high-value facts.
- Cards for grouped decisions, risks, or next steps.
- Dense but readable tables with repeated headers for print.
- Monospace code blocks with wrapping or horizontal containment.
- Footer with artifact date and provenance.

## Print CSS

Use an explicit page size, sensible margins, and `break-inside: avoid` on compact cards and rows where it improves continuity. Let long evidence sections break naturally. Do not shrink body text merely to force a page count.
